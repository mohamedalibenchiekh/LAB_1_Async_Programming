# Lab 1: Asynchronous Programming in Node.js

This laboratory explores asynchronous programming concepts in Node.js, demonstrating the evolution from traditional callbacks to modern async/await patterns.

## Overview

Asynchronous programming is crucial in Node.js for handling I/O operations without blocking the event loop. This lab covers different approaches to handle asynchronous operations, from basic callbacks to advanced async/await patterns.

## Files and Concepts

### 1. `Lab1_Callbacks.js`
- **Concept**: Traditional callback-based asynchronous programming
- **Features**:
  - File reading with callbacks
  - Error handling in callbacks
  - Callback hell demonstration
- **Run**: `node Lab1_Callbacks.js`

### 2. `Lab1_Promises.js`
- **Concept**: Promise-based asynchronous programming
- **Features**:
  - Converting callback-based APIs to promises
  - Promise states (pending, fulfilled, rejected)
  - Basic promise usage
- **Run**: `node Lab1_Promises.js`

### 3. `Lab1_Promises_Chaining.js`
- **Concept**: Promise chaining for sequential operations
- **Features**:
  - Chaining multiple asynchronous operations
  - Error propagation in promise chains
  - Sequential vs parallel execution
- **Run**: `node Lab1_Promises_Chaining.js`

### 4. `Lab1_AsyncAwait.js`
- **Concept**: Modern async/await syntax
- **Features**:
  - Converting promises to async/await
  - Synchronous-looking asynchronous code
  - Error handling with try/catch
- **Run**: `node Lab1_AsyncAwait.js`

### 5. `Lab1_Async_MultipleEvents.js`
- **Concept**: Handling multiple asynchronous operations
- **Features**:
  - Processing multiple files simultaneously
  - Promise.all() for parallel execution
  - Managing multiple async operations
- **Run**: `node Lab1_Async_MultipleEvents.js`

### 6. `Lab1_EventLoop.js`
- **Concept**: Understanding Node.js event loop
- **Features**:
  - Event loop phases demonstration
  - Blocking vs non-blocking operations
  - Event loop visualization
- **Run**: `node Lab1_EventLoop.js`

### 7. `Lab1_Performance.js`
- **Concept**: Performance comparison of different approaches
- **Features**:
  - Sequential vs parallel file reading
  - Performance benchmarking
  - Efficiency analysis
- **Run**: `node Lab1_Performance.js`

### 8. `Lab1_EventLogger.js`
- **Concept**: Complete real-world application
- **Features**:
  - Event management system
  - File I/O operations
  - Data analysis and logging
  - Complete async/await implementation
- **Run**: `node Lab1_EventLogger.js`

## Prerequisites

- Node.js installed (version 14 or higher recommended)
- Basic understanding of JavaScript

## How to Run

1. Clone or download this repository
2. Navigate to the Lab1 directory
3. Run individual files using `node <filename>.js`

Example:
```bash
cd Lab1
node Lab1_Callbacks.js
node Lab1_Promises.js
# ... run other files in order
```

## Learning Objectives

By the end of this lab, you will understand:

- The problems with callback-based asynchronous code

## Key Concepts Covered

- **Callbacks**: Traditional error-first callback pattern
- **Promises**: Modern promise-based approach
- **Async/Await**: Syntactic sugar over promises
- **Event Loop**: Node.js concurrency model
- **Parallel vs Sequential**: Performance considerations
- **Error Handling**: Different strategies for each pattern
